# ScanQR-WEB 操作说明书

## 一、系统简介

**ScanQR-WEB** 是一个微信小程序，用于制造车间的二维码扫码与流程卡管理系统。核心流程：管理员建卡 → 生成二维码 → 操作员扫码锁卡 → 填写工序数据 → 提交报工 → 全程可追溯。

### 角色体系

| 角色 | 权限范围 |
|------|----------|
| **系统管理员** (admin) | 全部功能：看板、卡片管理、日志、用户管理、角色管理、模板管理 |
| **操作员** (operator) | 仅扫码上报、填报流程卡 |
| **质检员** (qc) | 扫码上报、卡片追溯 |

可自定义创建新角色并分配权限。

---

## 二、首次使用 — 系统初始化

1. 用微信开发者工具打开项目，编译运行
2. 进入小程序后会自动跳转到**登录页**
3. 点击底部的 **"初始化系统"** 按钮
4. 初始化完成后自动填充管理员账号和密码：
   - **账号**: `admin`
   - **密码**: `admin123`
5. 点击 **"登录"** 即可进入系统

> 初始化操作会创建：管理员账号、三个内置角色、示例模板、测试卡片、数据字典。可重复执行（已存在的数据会跳过）。

---

## 三、登录

- 输入账号和密码，点击 **"登录"**
- 首次登录前必须先执行**系统初始化**
- 登录成功后，根据用户权限自动跳转：
  - 管理员 → **数据看板**
  - 操作员/质检员 → **扫码报到**

## 四、扫码报工（操作员日常操作）

### 4.1 获取工单号

进入 **"扫码"** 标签页，有 3 种方式获取工单号：

1. **扫描二维码**：点击扫码按钮，调用微信摄像头扫描二维码
2. **手动输入**：点击输入按钮，弹出对话框输入工单号
3. **上次工单**：自动记忆上次扫码的工单号

二维码由管理员建卡后在二维码生成页生成并打印，贴在工作现场。

### 4.2 锁定流程卡

扫码成功后，系统会调用 `getAndLockCard` 锁定该流程卡：
- 锁定期限：**30 分钟**（超时后其他人可抢占）
- 同一张卡同时只能被一人锁定，防止多人重复填写

### 4.3 填写流程卡

锁定成功后进入**流转卡填报页面**，包含以下内容：

**表头字段**（来自模板）：
- 项目名称、开单日期、交期日期、技术要求等

**工序明细**（来自模板，每道工序包含生产和品质两个部门）：
- 每道工序：设备编号、夹具号
- 生产部门：产出量、作业人员、作业完成时间、备注等
- 品质部门：检验数量、检验员、检验时间、不良原因等

**入库信息**（可选）：
- 入库人员、入库日期

**快捷操作**：
- 点击 `datetime` 类型字段 → 自动填充当前时间
- 点击 `select` 类型字段 → 弹出下拉选择框

### 4.4 提交 / 放弃

- **提交**：保存所有填写数据并解锁卡片
- **放弃**：仅解锁卡片，不保存数据
- 关闭页面时，如果未提交，系统**自动发送解锁请求**，防止锁死

## 五、二维码生成

1. 进入卡片管理页 → 点击 **"建卡"**
2. 填写工单号和工序信息，建卡成功
3. 在弹出的成功提示中点击 **"生成二维码"**
4. 跳转到二维码生成页，自动填入工单号
5. 点击 **"生成二维码"** 按钮，Canvas 绘制二维码
6. 可**保存到相册**，打印后贴到工作现场

也可以在 **"扫码"** 标签页中点击 "生成二维码" 入口进入此页面。

---

## 六、在制卡片管理

在 **"在制"** 标签页，可管理所有流程卡。

### 6.1 状态筛选

- **加工中**：正在进行的卡片
- **已完工**：已标记完工的卡片
- **已作废**：已作废的卡片

### 6.2 建卡

点击 **"建卡"** 按钮：
1. 选择流程卡模板
2. 输入工单号
3. 填写工序名称列表（每行一道工序，如：车削、铣削、磨削、电镀、包装）
4. 填写表头字段（项目名称、交期等）
5. 确认创建
6. 建卡成功后可选跳转**生成二维码**

### 6.3 卡片操作

点击卡片的操作按钮，可执行：

| 操作 | 说明 | 权限要求 |
|------|------|----------|
| 生命周期追溯 | 查看卡片所有操作记录 | `card_trace` |
| 强制解锁 | 释放被他人占用的锁 | `card_unlock` |
| 标记完工 | 将卡片状态改为已完工 | 管理员 |
| 作废 | 将卡片状态改为已作废 | 管理员 |
| 恢复加工 | 将作废/已完工恢复为加工中 | 管理员 |

## 八、操作日志

在 **"日志"** 标签页，查看所有操作记录。

### 8.1 日志筛选

- **分类筛选**：全部 / 正常提交 / 放弃 / 强制解锁
- **关键词搜索**：可按卡号、工单号、操作员、工序名称搜索
- **分页加载**：每页 20 条，上划自动加载更多

### 8.2 日志内容

每条日志包含：卡号、操作员、工序名称、状态标签（正常/放弃/强制解锁）、表单摘要、提交时间。

## 九、卡片追溯

在卡片管理页中点击 **"生命周期追溯"** 进入：
- 以**时间线**方式展示卡片的完整历史
- 每次提交的表单变更内容（字段级 before/after 对比）
- 不同状态用不同颜色标识（正常/放弃/强制解锁）

## 十、用户管理

在 **"我的"** 标签页 → **人员管理** 进入。

### 10.1 用户列表

显示所有用户的账号、姓名、部门、角色、状态（启用/禁用）、创建时间、最后登录时间。

### 10.2 新增用户

点击 **"新增用户"**，填写：账号、密码、姓名、部门、角色、电话。

### 10.3 用户操作

| 操作 | 说明 |
|------|------|
| 修改角色 | 变更用户的角色 |
| 重置密码 | 生成新密码，该用户所有登录态失效 |
| 启用/禁用 | 禁用后用户无法登录 |

> **保护机制**：系统确保至少保留一名管理员，最后的管理员不能被降级或禁用。

## 十一、角色管理

在 **"我的"** 标签页 → **角色管理** 进入。

### 11.1 权限模块

| 模块 | 权限点 | 说明 |
|------|--------|------|
| 看板 | `dashboard_view` | 查看数据看板 |
| 流程卡 | `card_list` | 查看在制卡片列表 |
| | `card_submit` | 扫码上报、填报流程卡 |
| | `card_unlock` | 强制解锁他人锁定的卡片 |
| | `card_trace` | 查看卡片生命周期追溯 |
| 日志 | `log_view` | 查看操作日志 |
| | `log_export` | 导出日志 |
| 人员 | `user_manage` | 管理用户账号 |
| 角色 | `role_manage` | 管理角色和权限 |
| 模板 | `template_manage` | 管理流程卡模板和字典 |

### 11.2 新建/编辑角色

- 填写角色标识（英文）、角色名称（中文）、描述

## 十二、模板管理

管理员可自定义流程卡模板，决定每类工单需要填写哪些字段。

### 12.1 表头字段

定义流程卡顶部信息，如项目名称、计划交期等。

### 12.2 明细字段

定义每道工序的填报内容，支持 5 种字段类型：

| 类型 | 说明 |
|------|------|
| text | 普通文本输入 |
| number | 数字输入 |
| select | 下拉选择（可绑定数据字典） |
| textarea | 多行文本域 |
| datetime | 点击自动记录当前时间 |

### 12.3 字段属性

- **变量名**：英文标识，格式为字母+数字+下划线
- **标签**：显示名称
- **必填**：是否必须填写
- **占位符**：输入框提示文字
- **默认值**：默认填充内容
- **选项列表**（select 类型）：下拉可选项，可绑定数据字典

### 12.4 数据字典

管理下拉选项的数据源，如不良原因、制程类型等。
- 字典标识（英文）+ 字典名称（中文）
- 选项列表（一行一个）

## 十四、常见问题

**Q: 登录失败怎么办？**
A: 确保已执行系统初始化。若仍失败，检查账号是否被禁用，或联系管理员重置密码。

**Q: 扫码后提示"卡片已被锁定"？**
A: 该卡已被其他操作员锁定填写中。等待对方提交后解锁，或联系管理员**强制解锁**。

**Q: 强制解锁后数据会丢失吗？**
A: 仅释放锁。如果前一个操作员已提交部分数据，解锁后数据仍然保留在卡片上。

**Q: TabBar 看不到某些标签？**
A: TabBar 根据用户权限动态显示。没有对应权限的标签不会显示（如操作员只看到"扫码"和"我的"）。

**Q: 如何更换模板？**
A: 在模板管理页新建模板并分配字段，卡片创建后模板即固定，不可更换。

**Q: 二维码可以重复扫描吗？**
A: 二维码包含工单号，对应的卡片如果未被锁定，每次扫码都会尝试锁定并填报。如果是已完工/已作废的卡片，扫码将无法锁定。
