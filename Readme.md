# ScanQR-WEB

微信小程序二维码扫码与流程卡管理系统，面向制造车间场景，覆盖建卡 → 扫码锁卡 → 工序填报 → 报工提交 → 全程追溯的完整链路。

## 技术栈

- **前端**: 微信小程序原生框架 (JavaScript)，TDesign Miniprogram 组件库
- **后端**: 微信云开发 CloudBase，云函数 (Node.js + wx-server-sdk)
- **渲染**: 支持 Skyline 渲染引擎
- **主题**: 支持深色模式 (light/dark)
- **国际化**: `i18n/base.json`

## 核心架构

### 鉴权体系

- 所有管理员页面入口调用 `auth.requireLogin()` 检查登录态
- 权限判断使用 `auth.hasPerm(perm)` 检查具体权限点
- 云函数调用使用 `auth.callWithAuth(name, data)` 自动注入 `session_token`
- 会话过期自动清除本地缓存并跳转登录页

### 自定义 TabBar

TabBar 根据用户角色权限动态显示/隐藏标签页，无权限的标签不出现：

| Tab | 权限点 | 说明 |
|-----|--------|------|
| 看板 | `dashboard_view` | 数据看板 |
| 在制 | `card_list` | 卡片管理 |
| 日志 | `log_view` | 操作日志 |
| 扫码 | `card_submit` | 扫码报工 |
| 我的 | 无限制 | 个人中心（始终可见）|

### 模板驱动低代码表单

流转卡表单页面 (`flow-card` / `process-form`) 作为低代码渲染引擎运行：从模板读取字段定义 (`header_fields` + `detail_fields`)，动态渲染 input、number、select、textarea、datetime 等 5 种控件类型，无需硬编码字段。

### 卡片锁定机制

扫码后通过 `getAndLockCard` 锁定流程卡（独占预约，锁定期 30 分钟），提交/放弃后自动解锁，页面关闭时通过 `onUnload` 兜底解锁，防止死锁。

## 一、系统简介

**ScanQR-WEB** 是一个微信小程序，用于制造车间的二维码扫码与流程卡管理系统。核心流程：管理员建卡 → 生成二维码 → 操作员扫码锁卡 → 填写工序数据 → 提交报工 → 全程可追溯。

### 角色体系

| 角色 | 权限范围 |
|------|----------|
| **系统管理员** (admin) | 全部功能：看板、卡片管理、日志、用户管理、角色管理、模板管理、工序字典 |
| **操作员** (operator) | 仅扫码上报、填报流程卡 |
| **质检员** (qc) | 扫码上报、卡片追溯 |

可自定义创建新角色并分配权限。

## 五、在制卡片管理

在 **"在制"** 标签页，可管理所有流程卡。

### 5.1 状态筛选

- **加工中**：正在进行的卡片
- **已完工**：已标记完工的卡片
- **已作废**：已作废的卡片

### 5.2 建卡

点击 **"建卡"** 按钮：

1. **选择模板**：点击"请选择模板"，从列表中选取所需模板
2. **输入工单号**（如模板未自带）：填写工单号
3. **选择工序**：从工序列表中**勾选**所需的工序（可多选，按选择顺序排列）
   - 已选工序以蓝色标签展示，显示序号
   - 点击标签上的 ✕ 可移除
   - 如需补充列表中不存在的工序，点击 **"自定义工序"** 手动输入添加
4. **填写表头字段**：项目名称、交期等
5. 点击 **"创建"**

> 工序列表由管理员在"工序与字典管理"中维护，建卡时直接勾选即可，减少手动输入。

### 5.3 卡片操作

点击卡片的操作按钮，可执行：

| 操作 | 说明 | 权限要求 |
|------|------|----------|
| 生命周期追溯 | 查看卡片所有操作记录 | `card_trace` |
| 强制解锁 | 释放被他人占用的锁 | `card_unlock` |
| 标记完工 | 将卡片状态改为已完工 | 管理员 |
| 作废 | 将卡片状态改为已作废 | 管理员 |
| 恢复加工 | 将作废/已完工恢复为加工中 | 管理员 |

## 六、扫码报工（操作员日常操作）

### 6.1 获取工单号

进入 **"扫码"** 标签页，有 3 种方式获取工单号：

1. **扫描二维码**：点击扫码按钮，调用微信摄像头扫描二维码
2. **手动输入**：点击输入按钮，弹出对话框输入工单号
3. **上次工单**：自动记忆上次扫码的工单号

### 6.2 锁定流程卡

扫码成功后，系统会调用 `getAndLockCard` 锁定该流程卡：
- 锁定期限：**30 分钟**（超时后其他人可抢占）
- 同一张卡同时只能被一人锁定

### 6.3 填写流程卡

锁定成功后进入**流转卡填报页面**，包含表头字段、工序明细（每道工序含生产和品质部门）、入库信息等。

**快捷操作**：
- 点击 `datetime` 类型字段 → 自动填充当前时间
- 点击 `select` 类型字段 → 弹出下拉选择框

### 6.4 提交 / 放弃

- **提交**：保存所有填写数据并解锁卡片
- **放弃**：仅解锁卡片，不保存数据
- 关闭页面时未提交则**自动解锁**

## 十四、常见问题

**Q: 登录失败怎么办？**
A: 确保已执行系统初始化。若仍失败，检查账号是否被禁用，或联系管理员重置密码。

**Q: 扫码后提示"卡片已被锁定"？**
A: 该卡已被其他操作员锁定。等待提交后解锁，或联系管理员**强制解锁**。

**Q: 建卡时找不到需要的工序？**
A: 请管理员在"工序与字典管理"中编辑"工序列表"字典，添加所需工序。或建卡时点击"自定义工序"手动添加。

**Q: TabBar 看不到某些标签？**
A: TabBar 根据用户权限动态显示，无权限的标签不会出现。

**Q: 二维码可以重复扫描吗？**
A: 二维码含工单号，对应卡片未被锁定时可扫码填报。已完工/作废的卡片无法再锁定。
